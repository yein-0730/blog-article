# 블로그 아티클 자동 작성 웹페이지 설계

> **날짜**: 2026년 3월 22일  
> **참여자**: B2B 마케터 (교육업) + Claude

---

## 1. 프로젝트 개요

**목적**: 클로드 코드 교육에서 시연할 "블로그 아티클 자동 작성 웹페이지" 설계 및 구현

**핵심 컨셉**: 시즌/트렌드 기반 주제 추천부터 HRD 타겟 B2B 블로그 아티클 완성까지 원스톱 웹앱

---

## 2. 요구사항 정의

| 항목 | 내용 |
|------|------|
| 타겟 독자 | HRD 담당자 (교육업 종사자) |
| 입력 방식 | 시즌/트렌드 주제 추천에서 선택 또는 직접 입력 |
| 핵심 기능 | 주제 추천 + 아티클 자동 생성 |
| 부가 기능 | SEO / GEO 최적화 제안, 시각화 초안 (이미지 프롬프트) |

### 4-2. AI 추천 조합 방식

시즈널 캘린더(정적) + Claude AI(동적)를 조합하여 주제를 추천합니다:

```
[주제 추천 프롬프트]

당신은 B2B 교육업 전문 콘텐츠 전략가입니다.

[현재 맥락]
- 현재 날짜: {currentDate}
- 이번 달 시즈널 키워드: {seasonalKeywords}
- 이번 달 HRD 맥락: {seasonalContext}
- 타겟 독자: {reader}

[요청]
위 시즈널 맥락과 최신 HRD/AX 트렌드를 결합하여
지금 이 시점에 {reader}가 가장 관심을 가질 블로그 주제 5개를 추천해주세요.

[출력 형식 - JSON]
{
  "topics": [
    {
      "title": "추천 주제 제목",
      "reason": "왜 지금 이 주제인지 1줄 설명",
      "angle": "이 주제를 다룰 때 차별화 포인트",
      "keywords": ["관련 키워드 2~3개"]
    }
  ]
}
```

- 설정 옵션
  - **톤**: 전문적·신뢰감 / 친근·실용적 / 도전적·인사이트형
  - **길이**: 800자 / 1500자 / 2500자
  - **독자 직무**: HRD 담당자 / CLO·교육팀장 / HR 기획자 / 교육 운영자

- 마크다운 복사

- → 티스토리, 워드프레스, 노션 등에 바로 붙여넣기 가능

## 6. Claude API 프롬프트 설계

### 6-1. 메인 생성 프롬프트 (System Prompt)

```
당신은 B2B 교육업 전문 콘텐츠 마케터입니다.
기업 교육(HRD), 인재개발, 조직학습 분야에 깊은 전문성을 갖고 있으며,
HRD 담당자가 실무에 바로 활용할 수 있는 인사이트 중심의 글을 작성합니다.

[핵심 원칙]
- 주장에는 반드시 근거를 제시하세요 (프레임워크, 구조적 분석, 검증 가능한 통계 등)
- 사례는 실제로 검증 가능한 것만 사용하세요:
  ✅ 허용: 공개된 보고서·논문의 통계 (출처 명시), 언론에 보도된 기업 사례, 공식 발표 자료
  ❌ 금지: 가상의 기업 사례, 만들어낸 수치, 출처 없는 "~한 기업에서는" 식 익명 사례
- 검증 가능한 사례가 없으면 사례 없이 프레임워크·구조·방법론의 깊이로 설득하세요
- 구체적 수치를 모르면 "업계 조사에 따르면" 등 일반적 표현을 사용하세요
- HRD 실무자가 "우리 회사에도 적용할 수 있겠다"고 느낄 수 있도록 실행 가능한 내용을 포함하세요
- 각 섹션이 독립적으로도 의미가 있되, 전체가 논리적으로 연결되어야 합니다
- 문장 간 연결이 자연스럽게 흐르도록 하세요. 접속어·전환 표현("그런데", "그래서", "이처럼", "반면")을 활용하여 뚝뚝 끊기는 느낌을 피하세요
```

### 6-2. 톤별 세부 지침

> 아래 지침은 기존 아티클 6편의 톤 패턴을 분석하여 도출한 것입니다.

**[공통 톤 원칙] — 모든 톤에 기본 적용**
```
[공통 톤 원칙]
- 서술체는 합쇼체(~입니다, ~됩니다)를 기본으로 사용하세요
- 독자를 직접 호명하지 않되, "우리 조직", "현장에서는" 등 공감 표현으로 거리를 좁히세요
- 문제 제기 → 원인 분석 → 구조적 해법 → 실행 제안의 4단 논리 흐름을 따르세요
- 추상적 주장 직후에 반드시 프레임워크·구조적 분석·검증 가능한 데이터를 배치하세요. 검증할 수 없는 가상 사례는 사용하지 마세요
- 섹션 말미에 다음 섹션으로의 논리적 브릿지 문장을 넣으세요 ("이 관점이 정리되면 질문은 ~로 바뀌어야 합니다")
- CTA(행동 유도)는 글 마무리에 자연스럽게 녹이되, 직접적 광고 톤은 피하세요
- 이모지는 👉, ✔, 📌 정도만 링크/체크리스트에 제한적으로 사용하세요
- 사용 금지 단어: "병목" — 대신 "구조적 한계", "정체 구간", "간극" 등으로 대체하세요

[AEO/GEO 최적화 구조 원칙]
AEO(Answer Engine Optimization): AI/검색엔진의 답변으로 직접 채택되기 위한 구조
GEO(Generative Engine Optimization): 생성형 AI 답변에서 출처로 인용되기 위한 구조

- 소제목은 질문형("왜 ~하는가", "어떻게 ~할 것인가")으로 작성하세요 (AEO: 사용자 질문과 매칭)
- 각 섹션의 첫 1~2문장에서 소제목 질문에 대한 직접 답변을 제공하세요 (AEO: Featured Snippet 채택 구간)
  예시) 소제목: "AI 기업교육이 성과로 이어지지 않는 이유는 무엇인가"
        첫 문장: "가장 큰 원인은 교육 전후를 연결하는 측정 구조가 설계되지 않았다는 점입니다."
- 핵심 개념에는 정의형 문장을 포함하세요 ("~란 ~을 의미합니다" / "~는 ~를 가리킵니다")
  → AI가 정의를 발췌할 때 우선적으로 참조하는 형식입니다 (GEO)
- 비교·분류 정보는 반드시 테이블 또는 번호 리스트로 구조화하세요
  → AI가 정리된 데이터를 우선 인용합니다 (GEO)
- 통계·데이터 인용 시 출처를 명확히 기재하세요 ("Deloitte 2025 보고서에 따르면")
  → AI가 신뢰할 수 있는 콘텐츠를 판별하는 핵심 기준입니다 (GEO)
- 아티클 하단에 FAQ 섹션(2~3개 Q&A)을 자동 생성하세요
  → 사용자가 질문형으로 검색할 때 AI가 직접 답변으로 채택하는 구간입니다 (AEO)
- 단계별 가이드("Step 1 → Step 2 → ...")를 포함하면 AI의 프로세스형 답변 인용 확률이 높아집니다
```

각 톤 옵션 선택 시 System Prompt에 추가되는 지침:

**전문적·신뢰감** (아티클 1, 2, 3, 5 패턴)
```
[톤 지침]
- 합쇼체 기반의 객관적 서술체를 사용하세요 (~입니다, ~때문입니다, ~수밖에 없습니다)
- 한 단락 안에 "현상 → 원인 → 시사점" 구조를 반복하세요
  예시: "매출은 제자리, 야근도 제자리인데 성과 보고만 매끈하게 만들 수는 없기 때문입니다. 결국 병목은 교육 운영 자체가 아니라..."
- 통계·보고서 인용 시 출처를 명시하되, 구체 수치를 모를 경우 "업계 조사에 따르면" 등으로 처리하세요
- 프레임워크·모델을 적극 활용하세요 (예: 커크패트릭 4단계, 70-20-10, COE 모델, 파일럿→복제→확산)
- 핵심 개념은 표(테이블)나 번호 리스트로 구조화하세요 (예: "실무 중심의 AI 교육 KPI 예시 — 1.리드타임 2.비용 3.품질")
- 각 섹션 도입부에서 이전 논의를 한 문장으로 요약한 뒤 새로운 논점을 제시하세요
- 적합 상황: AX 전략 아티클, 경영진 대상 보고 근거, 워크숍 요약
```

**친근·실용적** (아티클 4, 6 패턴)
```
[톤 지침]
- 합쇼체를 유지하되, "~인데요", "~이죠", "~있습니다" 등 부드러운 종결어미를 섞으세요
- 독자에게 질문을 던지며 시작하세요 ("그렇다면 ~에는 어떤 것들이 있을까요?")
- 설명 후 📌 관련 링크 또는 실제 사례를 바로 이어 붙이세요
- 체크리스트(✅), 단계별 가이드, 사례 박스를 적극 활용하세요
- 전문 용어 사용 시 괄호 안에 쉬운 설명을 추가하세요 (예: "LMS(학습관리시스템)")
- 문장 길이를 짧게 유지하고, 한 문단을 3~4문장 이내로 끊으세요
- 적합 상황: 교육 담당자 실무 가이드, 트렌드 정리, 체크리스트형 콘텐츠
```

**도전적·인사이트형** (아티클 1, 3의 도입부 패턴 확장)
```
[톤 지침]
- 냉정한 현실 점검으로 시작하세요 ("여기서 한 번은 냉정하게 점검할 필요가 있습니다")
- 통념을 뒤집는 질문을 던지세요 ("교육은 계속했는데 왜 변화는 더딜까요?", "결국 핵심은 ~이 아닙니다")
- "~이 아니라 ~이다" 대비 구조를 반복하세요
  예시: "결국 핵심은 교육을 더 많이 여는 일이 아니라, 현업 변화와 성과가 이어지도록 운영 구조를 설계하는 일입니다"
- 독자가 기존 관점을 재검토하게 만드는 역설적 사례를 활용하세요
- 결론에서 패러다임 전환을 명확히 제시하세요 ("질문은 '어떤 강의를 할까'가 아니라 '어떤 워크플로우를 먼저 바꿀까'로 바뀌어야 합니다")
- 적합 상황: 리더십 대상 인사이트, 컨퍼런스 발표 콘텐츠, 소셜 미디어 화제성 글
```

## 7. API 응답 구조 및 에러 처리

### 7-1. API 호출 방식

| 항목 | 설계 |
|------|------|
| 모델 | `claude-sonnet-4-6` |
| 호출 방식 | **스트리밍 (SSE)** — 사용자에게 생성 중 실시간 표시 |
| 응답 형식 | JSON (스트리밍 완료 후 파싱) |
| 최대 토큰 | 4096 (본문 생성) / 1024 (섹션 재생성) |
| Temperature | 0.7 (창의성과 일관성 균형) |

### 7-2. 스트리밍 UX 처리

```
1. 생성 시작 → "아티클을 작성하고 있습니다..." 로딩 표시
2. 스트리밍 중 → 텍스트가 타이핑되듯 실시간 표시 (raw text)
3. 스트리밍 완료 → JSON 파싱 → 탭별 UI에 구조화 렌더링
4. 파싱 실패 시 → raw text를 "본문" 탭에 그대로 표시 + 재시도 버튼
```

### 7-3. 에러 처리 시나리오

| 에러 유형 | 상태 코드 | 사용자 메시지 | 처리 |
|-----------|----------|-------------|------|
| 네트워크 오류 | - | "네트워크 연결을 확인해주세요" | 재시도 버튼 |
| API 키 오류 | 401 | "API 설정을 확인해주세요" | 설정 페이지 이동 |
| 요청 한도 초과 | 429 | "잠시 후 다시 시도해주세요 (약 {n}초)" | 자동 재시도 (지수 백오프) |
| 서버 오류 | 500 | "일시적인 오류입니다. 다시 시도해주세요" | 재시도 버튼 |
| JSON 파싱 실패 | - | (경고 없이) 원문 텍스트 표시 | 재생성 버튼 노출 |
| 응답 길이 초과 | - | "글이 잘렸습니다. 이어서 작성할까요?" | 이어쓰기 버튼 |

### 7-4. API 키 관리

- 프론트엔드 단독 앱 (서버리스) 기준 설계
- 사용자가 직접 Claude API 키를 입력 → `localStorage`에 저장
- API 키는 서버로 전송하지 않음 (브라우저에서 직접 API 호출)
- 입력 화면 상단에 API 키 설정 영역 배치

## 9. SEO / AEO / GEO 제안 로직

### 9-1. SEO 제안 기준

| 항목 | 생성 규칙 |
|------|----------|
| **metaTitle** | 주요 키워드를 앞쪽에 배치, 60자 이내, 파이프(｜)로 브랜드 구분 |
| **metaDesc** | 독자 페인포인트 + 글의 가치 제안, 155자 이내, 행동 유도 문구 포함 |
| **primaryKeyword** | 사용자 입력 키워드에서 검색 의도에 가장 가까운 1개 추출 |
| **secondaryKeywords** | 주요 키워드의 연관어·롱테일 키워드 3~5개 |

### 9-2. AEO (Answer Engine Optimization) 제안

AI/검색엔진이 답변을 생성할 때 우리 콘텐츠를 **직접 답변으로 채택**하기 위한 구조 제안:

```
aeoTips 생성 기준:

1. 질문-답변 매칭 구조
   - 소제목이 사용자의 실제 검색 질문과 매칭되는지 점검
   - 예) "AI 기업교육 성과 측정 방법" 검색 → 소제목 "AI 기업교육 성과는 어떻게 측정해야 하는가"

2. 직접 답변 첫 문장 (Featured Snippet 구간)
   - 각 섹션 첫 1~2문장이 소제목 질문에 대한 명확한 답변인지 점검
   - AI는 질문 직후의 첫 문장을 답변으로 발췌하는 경향이 강함
   - 예) ❌ "많은 기업이 고민하고 있습니다" → ✅ "가장 효과적인 방법은 리드타임·비용·품질 세 축으로 측정하는 것입니다"

3. FAQ 섹션 품질
   - 생성된 FAQ가 타겟 독자의 실제 검색 패턴과 일치하는지 점검
   - 답변이 2~3문장으로 간결하고 자족적인지 확인 (답변만 읽어도 의미가 통해야 함)
```

### 9-3. GEO (Generative Engine Optimization) 제안

생성형 AI(ChatGPT, Perplexity, Google AI Overview 등)가 답변을 만들 때 우리 콘텐츠를 **출처·참고 자료로 인용**하기 위한 신뢰·권위 강화 제안:

```
geoTips 생성 기준:

1. 신뢰 신호 (Trust Signal)
   - 통계·데이터 인용 시 출처가 명확히 기재되어 있는지 ("Deloitte 2025 보고서에 따르면")
   - AI는 출처가 명시된 콘텐츠를 우선적으로 참고하려는 경향이 강함

2. 정의형 문장 (Definition Statement)
   - 핵심 개념에 "~란 ~을 의미합니다" 형태의 정의 문장이 포함되어 있는지
   - AI가 개념 설명 답변을 생성할 때 가장 먼저 발췌하는 형식

3. 구조화된 데이터 (Structured Data)
   - 비교·분류 정보가 테이블 또는 번호 리스트로 정리되어 있는지
   - AI는 정리된 구조화 데이터를 비정형 텍스트보다 우선 인용함

4. 단계별 프로세스 (Step-by-step)
   - 방법론·가이드가 "Step 1 → Step 2 → ..." 형태로 구조화되어 있는지
   - AI의 "~하는 방법" 답변에서 인용 확률이 높아짐

5. 맥락의 명확성 (Contextual Clarity)
   - 각 문단이 독립적으로 읽혀도 맥락이 통하는지
   - AI는 문단 단위로 발췌하므로, 앞뒤 문맥 없이도 의미가 완결되어야 함
```

## 10. 내보내기 포맷별 변환 로직

### 10-1. 마크다운 변환

```markdown
# {title}

{intro}

## {section.heading}

{section.directAnswer}

{section.body}

(각 섹션 반복)

---

{outro}

---

## 자주 묻는 질문

**Q. {faq.question}**
A. {faq.answer}

(FAQ 반복)
```

- 본문 내 볼드, 리스트, 인용 등 마크다운은 그대로 유지
- 이미지 프롬프트는 마크다운에 포함하지 않음 (본문만)
- 복사 시 클립보드에 plain text로 복사

### 10-2. HTML 변환

```html
<article>
  <h1>{title}</h1>
  <p>{intro}</p>

  <h2>{section.heading}</h2>
  <p><strong>{section.directAnswer}</strong></p>
  <div>{section.body — 마크다운 → HTML 변환}</div>

  <!-- 각 섹션 반복 -->

  <hr />
  <p>{outro}</p>

  <section class="faq">
    <h2>자주 묻는 질문</h2>
    <h3>{faq.question}</h3>
    <p>{faq.answer}</p>
    <!-- FAQ 반복 -->
  </section>
</article>
```

- 마크다운 → HTML 변환 라이브러리 사용 (`marked` 또는 `react-markdown`)
- 인라인 스타일 없이 시맨틱 HTML로 출력 (티스토리·워드프레스 호환)
- 복사 시 클립보드에 `text/html` + `text/plain` 동시 복사 (붙여넣기 시 서식 유지)

### 10-3. 복사 동작 명세

| 버튼 | 클립보드 포맷 | 복사 범위 | 완료 피드백 |
|------|-------------|----------|------------|
| 📋 마크다운 복사 | `text/plain` | 본문 전체 (SEO·시각화 제외) | "복사되었습니다 ✓" 토스트 2초 |
| 📋 HTML 복사 | `text/html` + `text/plain` | 본문 전체 (SEO·시각화 제외) | "복사되었습니다 ✓" 토스트 2초 |
| 📋 SEO 복사 | `text/plain` | meta title + desc + 키워드 | "복사되었습니다 ✓" 토스트 2초 |
